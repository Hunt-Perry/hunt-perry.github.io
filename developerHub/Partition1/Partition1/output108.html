Introduction

This is documentation of the API capabilites. This api can be consumed currently only via the node-agent-sdk

Happy Flow

  1. An agent creates a connection using username and password.
  2. The agent subscribes to get notification rings on new conversation tasks from the routing service (subscribeRoutingTasks).
  3. The agent subscribes to get his list of existing assigned conversations (subscribeExConversations).
  4. The agent sets his availablity to online (setAgentState)
  5. The agent gets a ring notifying on a new conversation task. (routingTaskNotification)
  6. The agent send his acceptance to get the task (updateRoutingTask).
  7. The agent get a notification on the new conversation added to his list of assigned conversations. (ExConversationNotification)
  8. The agent subscribes to get the content of this conversation. (subscribeMessagingEvents)
  9. The agent gets existing content events, and possibly other realtime notification about messages events that are being published to the conversation. (messagingEventNotification)
  10. The agent publish his own text messages, recipt events and presence events to the conversations. (publishEvent)
  11. The agent can remove his assignment to the conversation sending it back to the queue, or close it. (updateConversationField)

Subscribe Conversations Metadata

Subscribes to any changes in the metadata of the conversations of this user. Note:

  1. metadata is fetched for the first 1000 conversations that were viewed or changed (ascending by date). After that, changes for any conversation will be provided on an on-going basis.
  2. Notification order is not guaranteed. If notification order is needed in your application, use the timestamp supplied for each notification and ignore notifications with an older timestamp. You can use the metaDataLastUpdateTs parameter to do check for the notification timestamp and filter according to it.

Request Body

minLastUpdatedTime null
integer >= 0

Do not fetch conversation which have not been updated after this timestamp
convState null
string
"OPEN" "CLOSE"
